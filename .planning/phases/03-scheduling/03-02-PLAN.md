Create `scripts/setup-scheduler.sh` following the exact style and structure of `scripts/setup-secrets.sh`.

The script provisions three resources in order:

**Section 1: Prerequisites**
- Check PROJECT_ID env var (required)
- Check CLOUD_RUN_SERVICE_URL env var (required — the full URL of the Cloud Run service, e.g. https://network-data-validation-xxxxx-uc.a.run.app)
- Check gcloud CLI available
- Run `gcloud config set project "$PROJECT_ID"`
- Check Cloud Scheduler API is enabled: `gcloud services list --enabled --filter=cloudscheduler.googleapis.com`; if not found, enable it: `gcloud services enable cloudscheduler.googleapis.com`

**Section 2: Service Account**
Service account name: `network-data-scheduler-sa`
Display name: "Network Data Validation Scheduler"

Idempotent check-before-create:
```bash
SA_EMAIL="network-data-scheduler-sa@${PROJECT_ID}.iam.gserviceaccount.com"

if gcloud iam service-accounts describe "$SA_EMAIL" --project="$PROJECT_ID" &>/dev/null; then
  echo "  Service account already exists: $SA_EMAIL — skipping"
else
  echo "  Creating service account: network-data-scheduler-sa"
  gcloud iam service-accounts create network-data-scheduler-sa \
    --display-name="Network Data Validation Scheduler" \
    --project="$PROJECT_ID"
fi
```

**Section 3: IAM Binding**
Grant `roles/run.invoker` to the scheduler service account on the Cloud Run service.
Use `--member="serviceAccount:$SA_EMAIL"` and `--role="roles/run.invoker"`.
IAM bindings are idempotent by nature (gcloud silently skips duplicates), so no check needed:
```bash
echo "  Granting roles/run.invoker to $SA_EMAIL on Cloud Run service..."
gcloud run services add-iam-policy-binding network-data-validation \
  --region=us-central1 \
  --member="serviceAccount:$SA_EMAIL" \
  --role="roles/run.invoker" \
  --project="$PROJECT_ID"
```

**Section 4: Cloud Scheduler Job**
Job name: `network-data-validation-scheduler`
Idempotent check-before-create (if exists, update; if not, create):

```bash
JOB_NAME="network-data-validation-scheduler"

if gcloud scheduler jobs describe "$JOB_NAME" --location=us-central1 --project="$PROJECT_ID" &>/dev/null; then
  echo "  Updating existing Cloud Scheduler job: $JOB_NAME"
  gcloud scheduler jobs update http "$JOB_NAME" \
    --location=us-central1 \
    --schedule="0 */3 * * *" \
    --uri="${CLOUD_RUN_SERVICE_URL}/validate" \
    --http-method=POST \
    --oidc-service-account-email="$SA_EMAIL" \
    --oidc-token-audience="${CLOUD_RUN_SERVICE_URL}" \
    --max-retry-attempts=5 \
    --min-backoff=1m \
    --max-backoff=16m \
    --attempt-deadline=30m \
    --time-zone=UTC \
    --project="$PROJECT_ID"
else
  echo "  Creating Cloud Scheduler job: $JOB_NAME"
  gcloud scheduler jobs create http "$JOB_NAME" \
    --location=us-central1 \
    --schedule="0 */3 * * *" \
    --uri="${CLOUD_RUN_SERVICE_URL}/validate" \
    --http-method=POST \
    --oidc-service-account-email="$SA_EMAIL" \
    --oidc-token-audience="${CLOUD_RUN_SERVICE_URL}" \
    --max-retry-attempts=5 \
    --min-backoff=1m \
    --max-backoff=16m \
    --attempt-deadline=30m \
    --time-zone=UTC \
    --project="$PROJECT_ID"
fi
```
