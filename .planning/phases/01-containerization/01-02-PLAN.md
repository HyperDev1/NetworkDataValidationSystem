<task type="auto">
  <name>Task 1: Create docker-compose.yml for local testing</name>
  <files>docker-compose.yml</files>
  <action>
Create `docker-compose.yml` in the project root for convenient local development and testing.

```yaml
services:
  validation:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    environment:
      - PORT=8080
    volumes:
      - ./config.yaml:/app/config.yaml:ro
      - ./credentials:/app/credentials:ro
    restart: "no"
```

**Key decisions per CONTEXT.md:**
- Volume mount `./config.yaml:/app/config.yaml:ro` — config is NOT baked into the image, mounted at runtime (read-only)
- Volume mount `./credentials:/app/credentials:ro` — OAuth tokens and service accounts mounted (read-only)
- Port 8080 mapped to host 8080
- `restart: "no"` — local testing, not a daemon
- Single service named `validation`
- No networks, no extra complexity — simple enough to `docker compose up` and have it work

**Do NOT add:**
- Health check in compose (Docker HEALTHCHECK in Dockerfile is sufficient for local)
- Multiple services
- Environment files (.env) — keep it explicit
