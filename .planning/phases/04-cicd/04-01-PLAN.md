The script must:

1. **Header** — describe purpose and usage (3 env vars required: PROJECT_ID, GITHUB_REPO, GCP_PROJECT_NUMBER)

2. **Prereqs** — validate PROJECT_ID, GITHUB_REPO, GCP_PROJECT_NUMBER are set; check gcloud present

3. **Enable APIs** — enable artifactregistry.googleapis.com and iamcredentials.googleapis.com (check-before-enable pattern from setup-scheduler.sh)

4. **Artifact Registry** (section `=== Artifact Registry ===`)
   - Check if repository `network-data-validation` exists in us-central1
   - If not: `gcloud artifacts repositories create network-data-validation --repository-format=DOCKER --location=us-central1 --description="Network Data Validation System images"`
   - If yes: echo "Already exists — skipping"

5. **Service Account** (section `=== CI/CD Service Account ===`)
   - SA name: `network-data-cicd-sa`
   - Check-before-create: `gcloud iam service-accounts describe network-data-cicd-sa@$PROJECT_ID.iam.gserviceaccount.com`
   - Grant three roles (always run, idempotent):
     - `roles/artifactregistry.writer` on project
     - `roles/run.admin` on project
     - `roles/iam.serviceAccountTokenCreator` on the SA itself (self-binding for impersonation)

6. **Workload Identity Pool** (section `=== Workload Identity Pool ===`)
   - Pool name: `github-actions-pool`
   - Check-before-create pattern
   - `gcloud iam workload-identity-pools create github-actions-pool --location=global --display-name="GitHub Actions Pool"`

7. **Workload Identity Provider** (section `=== Workload Identity Provider ===`)
   - Provider name: `github-provider`
   - Check-before-create pattern (describe the provider before creating)
   - Attributes: `--attribute-mapping="google.subject=assertion.sub,attribute.repository=assertion.repository,attribute.actor=assertion.actor,attribute.ref=assertion.ref"`
   - Issuer: `--issuer-uri=https://token.actions.githubusercontent.com`
   - Condition: `--attribute-condition="assertion.repository=='$GITHUB_REPO'"`

8. **IAM Binding** (section `=== Workload Identity Binding ===`)
   - Bind the WI principal to the SA using principalSet (not principal, to allow all tokens from the repo):
     `--member="principalSet://iam.googleapis.com/projects/$GCP_PROJECT_NUMBER/locations/global/workloadIdentityPools/github-actions-pool/attribute.repository/$GITHUB_REPO"`
   - Role: `roles/iam.workloadIdentityUser` on the service account
   - Always run (IAM bindings are idempotent by design)

9. **Summary** (section `=== Setup Complete ===`)
   - Print the three values operators must add as GitHub repository secrets:
     - `WIF_PROVIDER`: `projects/$GCP_PROJECT_NUMBER/locations/global/workloadIdentityPools/github-actions-pool/providers/github-provider`
     - `WIF_SERVICE_ACCOUNT`: `network-data-cicd-sa@$PROJECT_ID.iam.gserviceaccount.com`
     - `GCP_PROJECT_ID`: `$PROJECT_ID`
   - Include clear instructions: "Add these as GitHub repository secrets (Settings → Secrets and variables → Actions)"
